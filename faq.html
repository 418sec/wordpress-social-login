A If you want to translate this plugin and you are new to WP/i18n, then we recommend to watch this video: Localizing and Translating WordPress Plugins.

The location of languages folder is /wordpress-social-login/languages/.

Note: WSL uses _wsl_e() instead of _e(), and _wsl__ instead of __().

Q I'm having "Your server failed the requirements check for this plugin!". What does it mean?
A In order for WordPress Social Login to work properly, your server should meet certain requirements. These "requirements" and "services" are usually offered by default by most "modern" web hosting providers, however some complications may occur with shared hosting and, or custom wordpress installations.

To determine what may cause this failure, you need to run the WordPress Social Login Requirements Test, after that, and on most cases, the solution is to make a trouble ticket with your web host.

Q All the requirement are met, yet the plugin still not working. What do I do?
A Keep reading this FAQ and look at wordpress forum to see if the issue you have have been resolved, if not you may ask for support.

Q I'm having an infinite loop of redirection when using the popup for authentication
A This can be casued when "PHP Register Globals" are On.

The solution is to make a trouble ticket with your web host to disable it, or, if you have a dedicated server and you know what are you doing then edit php.ini file and turn it Off.

Q After I authenticate with a social network I'm redirected to white page, or getting 500 HTTP Error.
A Displaying blank page is mostly caused by some PHP error in the code. If error reporting is on (displaying PHP errors is enabled) then it is very easy to find the error and solve the problem.

Q I'm getting 404 HTTP Error page not found
A If you get a 403 and 404 on the Rewrite Diagnostics test, request your hosting provider whitelist your domain on mod_security. This problem has been encountered with Host Gator and GoDaddy.

Q "You cannot access this page directly." Error
A Most likely an issue with PHP SESSION.

WSL requires a PHP's default SESSION setup in order to works; and this issue could occur due to the following:

  • Your webhost have purposely disabled PHP's SESSION. Eg: WP Engine.
  • PHP's SESSION have been renamed php.net/manual/en/function.session-name.php.
  • File permissions stackoverflow.com/questions/ask.
  • If you are using a reverse proxy like Varnish it is possible that WordPress's default user cookies are being stripped

Q When I click on the icons nothing happen.
A Goto WordPress Social Login > Widget > Advanced Settings and change "Authentication flow" to "No popup window".

Note: In WSL 2.2.1+, "Authentication flow" is renamed to "Authentication display", and "No popup window" is renamed "Popup".

Q Why Twitter users email look like this: twitter_user_<something>@example.com?
A Some social networks like Twitter and LinkedIn do NOT provide their user's email address and by default a random email will then be generated for them instead. To change this behaviour and to force new registered users to provide their emails before they get in, goto Bouncer and enable Email Validation.

Q Everything works except Facebook, when using it I got a PHP error, white page or HTTP 500 error.
A This usually happen when an another plugin (related to facebook) is conflicting with WordPress Social Login.

To make sure if it's the case, disable all of them and try to authenticate with facebook once again. The solution is to ask for support from the plugins authors you are using.
