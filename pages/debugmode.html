Debug mode

Debug mode is an internal developement tool built to track every action made by WSL during the authentication process. The debug mode can be useful when debugging this plugin but note that it is highly technical and not documented. When Debug mode is enabled and set to Log actions in a file, WSL will attempt to generate its log files under /wp-content/uploads/wordpress-social-login .

Important

It's recommended to keep the Debug Mode Disabled as will store unnecessary data on your web server. Please, don't use Debug mode unless:

  • You are studing how WSL works
  • You are implementing new things into WSL
  • You have an issue and you really really want WSL to work
