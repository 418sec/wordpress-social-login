Filters Developer API

In WSL 2.2.1, all the hooks has been reworked and few hooks have been depreciated in favour of new ones. Please update the WSL hooks you were using yours accordingly to this new documentation.

wsl_hook_alter_provider_scope

Allow users to overwrite providers scopes. 

$provider_scope = apply_filters( 'wsl_hook_alter_provider_scope', $provider_scope, $provider );

For an example, we can redefine the number of permissons to ask of facebook users: 

function wsl_lower_facebook_permissons( $provider_scope, $provider )
{
    if( 'facebook' == strtolower( $provider ) ) // < strtolower
    {
        # https://developers.facebook.com/docs/facebook-login/permissions
        $provider_scope = 'email'; // should not be empty or it will be overwritten
    }
    
    return $provider_scope;
}

add_filter( 'wsl_hook_alter_provider_scope', 'wsl_lower_facebook_permissons', 10, 2 );

wsl_hook_process_login_alter_wp_insert_user_data

Change the wordpress user data before creating it with wp_insert_user. 

$userdata = apply_filters( 'wsl_hook_process_login_alter_wp_insert_user_data', $userdata, $provider, $hybridauth_profile );

wsl_hook_process_login_delegate_wp_insert_user

This action runs just before creating a new wordpress user, it delegate wp_insert_user to a custom function. 

$user_id = apply_filters( 'wsl_hook_process_login_delegate_wp_insert_user', $userdata, $provider, $hybridauth_profile );

wsl_render_login_form_alter_provider_icon_markup

Customize the HTML generated for each provider on WSL widget. 

$user_id = apply_filters( 'wsl_render_login_form_alter_provider_icon_markup', $provider_id, $provider_name, $authenticate_url );

For an example, we can display Font Awesome icons intead of the default ones.

Notes: In order for the buttons to display correctly in this example, Font Awesome should be integrated into your website theme. 

function wsl_use_fontawesome_icons( $provider_id, $provider_name, $authenticate_url )
{
   ?>
   <a 
      rel           = "nofollow"
      href          = "<?php echo $authenticate_url; ?>"
      data-provider = "<?php echo $provider_id ?>"
      class         = "wp-social-login-provider wp-social-login-provider-<?php echo strtolower( $provider_id ); ?> btn btn-<?php echo strtolower( $provider_id ); ?>" 
    >
      <span>
         <i class="fa fa-<?php echo strtolower( $provider_id ); ?>"></i>
         Sign in with <?php echo $provider_name; ?>
      </span>
   </a>
<?php
}

add_filter( 'wsl_render_login_form_alter_provider_icon_markup', 'wsl_use_fontawesome_icons', 10, 3 );
See this hook on github »
